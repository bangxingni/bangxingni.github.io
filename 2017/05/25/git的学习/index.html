git总结

转载、有改动

git和svn的区别?
git有本地仓库和缓存区

缓存区：只能使用git gui和git shell查看,提交代码解决 冲突的中转站。
本地仓库:只能git shell显示 ，本地和远程代码的枢纽。不能 联网时本地的代码可以先放在本地仓库
远程仓库：例如github
工作区间：创建工程文件的地方,编辑器可直接查看

一、 提交代码到远程仓库

（依次输入以下指令）
1、在工作区间中创建一个项目叫motherfar，并且项目里创建一个readme.md文件
2、github上创建工程 (复制网址)
3、git init 在当前项目目录中生成本地git管理
git add Readme.md（将这个文件放到缓冲区） 或者git add -A(将新增、修改、删除的文件全部放到缓冲区–>实际开发中的做法)-
git commit -m "随便起个名字"（代码从缓冲区放到本地仓库）或者git commit -am “说明文字”(a是all的意思，将缓冲区所有改动放到本地仓库–>实际开发中的做法)
4、git remote add origin http://github.com/......git 本地仓库与远程仓库建立联系。
5、push -u origin master 本地仓库代码给远程仓库（输入指令后会要求输入github帐号和密码,-u是用来指定默认主机，第一次提交需要输入）

二、将远程仓库代码提交到本地

1、新建一个文件夹（随便起名比如copycopy）
2、复制放到本地仓库上的网址在git bush中输入 git clone https://......//motherfar 远程仓库--》本地仓库(工作区间中文件与本地仓库是一致的，这时候缓冲区是空白) 
这时候copycpoy文件夹底下有了mothership

三、更新到本地仓库报错

nocopy中修改README.md文件，提交修改到远程仓库
copycopy/motherfar中修改README.md 文件，提交前将远程仓库的代码更新到本地仓库–>git pull会报错
因为本地工作区间和远程仓库的代码有冲突
有两种解决办法–》1、放弃本地修改2、解决冲突后提交本地修改
放弃本地修改：远程仓库代码覆盖本地仓库和工作区间
小白向->把本地工程全部删除重新clone
菜鸟向-> git checkout head . (这个.很重要的)
之后再次更新远程仓库的代码就不会有问题了

老鸟向->老鸟们会用更高级的指令来解决这种指令冲突，在git中还存在上面描述的第5个区叫做
->dirty working direcoty (工作现场)
使用git stash操作将工作区和缓存区的改动放到工作现场，工作现场独立于本地仓库和工作区
老鸟思路：本地修改后的代码放入工作现场，从远程仓库获取最新代码,完毕后从工作现场将代码取出，最新代码和修改后的代码会混杂在一起，手工解决冲突后可不报错的提交冲突后的代码。
